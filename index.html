项目简介

项目名称：大小屏联动教学解决方案

1. 项目定位与目标

本项目基于前期试点机构的调研结果，旨在解决传统课堂“教学过程黑盒化”与“设备管理失控”的痛点。

Demo 1.0 目标 快速验证“强管控下的高频互动”这一核心闭环在实际教学场景中的可行性。

Demo 核心逻辑：

  • 联动模式：老师大屏发起互动 → 学生 Pad 瞬间解锁 → 提交数据 → 自动锁定。
  • 数据采集：实现“人-题-轨迹”的数据采集。

第二模块：产品原型说明

第一步

默认不可操作，只有老师发起互动后才可操作。
1. 该 Demo 版本，基于安卓 Pad 开发一个单独的 App，且 App 中只有教室上课页面。
2. 学生端 APP 名称为「ClassIn Focus」。
3. 学生端的安卓 Pad 与教师端大屏通过某种方式建立关联关系，当教师进入教室后发起签到，学生的 Pad 端自动弹出签到页面。
4. 大屏没有进入课堂时，学生端启动 App 后，默认进入教室上课页面（线下模式，关闭麦克风摄像头）。提示：静候开课。你的智慧学伴已准备就绪。

第二步

老师进入教室后学生端显示名单，点击完成签到并绑定设备账号。老师关闭签到后回到静默状态。

产品逻辑说明：学生设备绑定方案对比矩阵
结合当前 Demo 1.0 的"强管控 + 高频互动"目标，从交互、风控、执行成本三维度评估签到 / 设备绑定方案的取舍。
方案 交互逻辑 核心优点 风险与行为边界 实际执行复杂度
1. 列表签到 老师下发名单，学生在 Pad 列表点击头像认领。 操作零门槛：极速开课，无需记忆，适合低幼学生。（当前xes用的模式） 数据污染风险：极易发生误点或代签。需处理"身份抢占"逻辑（如 A 占了 B 的位置）。 低
2. 账密登录 学生手动输入完整手机号与密码。 确权最严谨：符合标准互联网安全逻辑，身份唯一。 流程断裂：学生（尤其是低年级）记不住账号。Demo 1.0 阶段会极大增加现场维护成本。 中
3. 尾号验证 点击头像 + 输入家长手机号后 4 位。 平衡性最佳：通过双重复核防止误点，隐私压力小，利用现有数据库即可。 操作冲突处理：需设定为"先选人，后验号"，以规避不同家长手机尾号相同的物理碰撞。但要考虑学生能否记住家长后四位号码。 中
4. 人脸识别 摄像头自动扫脸，匹配云端底片绑定。 极致 AI 体验：100% 真实数据，防代签，符合"智慧教育"高端定位。 法律合规风险：未成年肖像权极其敏感；需提前采集底片；受光线等物理环境影响大。 高

深度诊断总结

👤 学生画像：学霸型选手

态度积极（100%参与），基础雄厚（简单/中等题近乎全对）。当前瓶颈在于建模思维稍弱，习惯心算而非动手画图。

建模思维瓶颈

遇到复杂相遇或几何切割题，习惯心算，缺乏画线段图或建立空间模型的习惯。

审题惯性思维

对常见题型反应快，但对陷阱（如单位不统一、求表面积还是侧面积）警惕性不够。

立体图形综合应用
