课堂报告服务

根据班级与课节数据，为每位学生生成课堂学习记录（V1）与深度诊断报告（V2）。

数据说明
  1. 生成报告前需提供 班级 ID、课节 ID（classId、lessonId）。
  2. 系统按 ID 查询并拉取：班级名称、课节名称、学生名单，以及每位学生在本节课的课堂过程数据。
  3. 课堂数据按时间线记录，包括各时间点的 互动答题器（选项、对错）与 小黑板图片（一图一题）。
  4. 基于上述结构化数据，自动生成各学生的 V1 学习记录与 V2 诊断报告。

上方可按班级 ID / 班级名称搜索已完课课堂并查看学生报告；页面底部「演示工具」仍支持 JSON / Word 导入，用于调试报告样式与 AI 解析。

PRODUCT ROLLOUT

产品化方案 · 把课堂报告交到一线手里

本页面是「课堂报告」生成内核的原型与调试入口。要让教师、教研、家长无阻碍用上，规划以下 两条可并行落地的产品化路径，共用同一套 V1 / V2 生成内核与数据接口，仅在入口与交互形态上有所区别。

1

机构后台 · 教师发展独立页面

沉淀型入口 —— 可查、可改、可分享
★ 推荐主线 · 标准产品路径
  • 落地位置：机构后台 教师发展 模块新增二级菜单「课堂报告」。
  • 核心能力：按 班级 / 教师 / 时间 查询已完课课堂，一键生成 V1 学习记录 + V2 深度诊断报告。
  • 报告形态：原生 HTML 页面，生成后自动落库保存，支持二次查看与按需重生成。
  • 分享方式：每份学生报告生成独立链接
  • 权限控制：大后台 版本配置 按机构维度控制该菜单（现有能力，增加开关）
2

客户端 AgentIn · 对话式 Agent

触达型入口 —— 下课即用、当场就有
⚡ 补充入口 · 即时触达
  • 落地位置：将能力封装为 课堂报告 Agent，集成进客户端 AgentIn 入口。
  • 触发方式：教师在对话框直接发起，如「生成今天五年级 3 班数学课的报告」即可调用。
  • 输出形态：Agent 在对话流内返回 HTML 报告卡片或链接，支持对话内预览与跳转独立分享链接。
  • 协同价值：无需切换到后台网页，教师下课/换场顺手即可触发，同时为后续「AI 教学助手」沉淀使用入口。
  • 权限沿用：报告权限、分享策略与方案 1 完全一致，结果可在后台「课堂报告」页面回查。

两者关系：同一生成内核 + 同一数据空间，仅交付通路不同。方案 1 是结构化的"沉淀型"产品页，承担归档、回查、对外分享；方案 2 是流式"触达型"入口，承担即时生成、轻量分发。建议方案 1 先行落地、方案 2 紧随集成，两条入口共享报告库，避免双套数据。
